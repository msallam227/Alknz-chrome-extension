Extension Features & Behavior Rules

This page documents exactly how each feature behaves inside the ALKNZ Intelligence Chrome Extension. These rules are deterministic and designed to prevent data corruption, duplication, and overwrites.

Capture Intelligence

Capture is the entry point of the extension. It scrapes visible and structured information from the active browser tab and prepares it for enrichment.

What is captured
  • Detected entity name (person or organization)
  • Job titles, roles, and affiliations
  • Public contact signals (email, phone if visible)
  • Geographic indicators (country, city, region)
  • Source URL and page metadata
What is NOT captured
  • Private or authenticated content
  • Hidden DOM elements
  • Images, PDFs, or file attachments
  • Anything behind paywalls
Capture does not write to the backend. It only prepares structured data locally.

Deep AI Enrichment

Deep AI analyzes captured text to extract normalized fields, classifications, and summaries. This step is optional and only runs when enabled.

Extracts
  • Clean entity name
  • Normalized role/title
  • Firm classification
  • Sector hints
Generates
  • Short descriptive summary
  • Relevance signals
  • Confidence indicators
Does NOT
  • Overwrite existing backend data
  • Create records automatically
  • Make investment recommendations
If Deep AI returns an empty or null extraction, the extension proceeds without enrichment.

Merge Rules (Critical)

Merge logic determines how captured and enriched data is applied to existing records. This logic is intentionally conservative.

Allowed merges
  • Only fill empty fields (setIfEmpty)
  • Union and deduplicate emails/phones (max 5)
  • Append new sources without removing old ones
Blocked behavior
  • Never overwrite populated fields
  • Never delete backend values
  • No silent auto-merges
The UI explicitly tracks whether a merge occurred (merged_into_form = true/false).

Upload to Backend

Upload sends the final payload to the backend server using the configured API key.

Payload includes
  • Captured fields
  • Deep AI enrichment (if used)
  • Merge flags and metadata
  • Source URL and timestamps
Backend response
  • Success or error status
  • Created or updated record ID
  • Validation errors (if any)
Upload is the only step that mutates backend data.

Feature Flow Summary

Page → Capture → (Optional) Deep AI → Controlled Merge → Upload. Every step is explicit, reversible, and designed to preserve data integrity.
